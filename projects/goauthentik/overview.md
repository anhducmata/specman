# Goauthentik — Overview

**Language:** TypeScript (Pulumi IaC)
**Purpose:** Provisions the Authentik identity provider on AWS. Manages SSO, user authentication, and Microsoft Entra ID (Azure AD) integration for the Amili platform.

---

## What It Does

Infrastructure-as-Code that deploys:
- Authentik server (identity provider) on ECS Fargate
- RDS PostgreSQL (Authentik database)
- ElastiCache Redis (Authentik cache/sessions)
- ALB (HTTPS termination, health checks)
- Route53 DNS + ACM TLS
- SSM Parameter Store (all secrets)

## Environments

| | Dev | Prod |
|---|---|---|
| URL | `https://auth.dev.amili.asia` | `https://auth.amili.asia` |
| ECS replicas | 1 | 2 |
| RDS Multi-AZ | No | Yes |
| Redis replicas | 1 | 2 |
| Deploy | Local or GitHub Actions | **GitHub Actions only** |

## Key Integration

Microsoft Entra ID (Azure AD) SSO for Amili staff login. Redirect URI: `https://auth.<env>.amili.asia/source/oauth/callback/azuread/`

## Secrets

All secrets generated by Pulumi, stored as `SecureString` in SSM. ECS tasks pull at runtime. Nothing hardcoded in task definitions.
